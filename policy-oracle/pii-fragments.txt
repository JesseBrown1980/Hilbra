# Hilbra public-tier PII deny-list — the 45-fragment policy oracle.
#
# These are DETECTION PATTERNS, not data. A path or row whose (lowercased) path or content
# contains any of these substrings classifies as OWNER-PRIVATE and is NEVER served on the
# public tier (level 0). This file is the conformance contract: every Hilbra node MUST run a
# byte-identical copy, and the public tier is provably PII-free ONLY if it does.
#
# Rules: PII match wins before any public-canon allow. Conservative by design — a false positive
# only over-privatizes a row (safe); a false negative would leak. Lines starting with # are comments.
# The conformance harness pins this file's sha256; CI guards against accidental gutting (>=40 lines).
#
# CONFORMANCE (binding): a node MUST test ALL 45 fragments against BOTH the lowercased path AND
# the lowercased content of every row (plus the >=14-consecutive-digit rule). Applying the 45 to
# the path only — with a smaller separate content list — UNDER-classifies and is NON-CONFORMANT.
# KNOWN GAP (2026-06-23, measured independently by acer + liris): the current recall engines
# (serve-recall.cjs and the Rust recall-serve, PR #8) apply these 45 to the PATH only, plus a
# separate 8-fragment content list. Until they apply all 45 to content too, those engines are
# NOT yet byte-conformant to this contract — tracked for hardening; PR #8 stays HELD.
#
